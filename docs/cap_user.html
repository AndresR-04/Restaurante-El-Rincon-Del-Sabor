Manual de capacitación


Introducción

El sistema Gestión Administrativa para el Registro de Ventas (Garvsoft), tiene funcionales principales como lo son: El registro de ventas, realizar ventas, registro de clientes, registro de platos y ver registro de ventas en formato PDF. El plan de capacitación al usuario está pensado en brindarle una ágil y eficiente información al usuario de tal manera en que pueda manejar el sistema informático sin ningún problema, por ello se plantea un estructuramiento inicial con resultados finales, como lo es:

  • Explicación detallada y general del aplicativo.
  • Explicación detallada de cada módulo el cual contiene el aplicativo.
  • Explicación de los manuales de usuario que se disponen para los usuarios, evitando algún inconveniente al usar el aplicativo.
  • Explicación de las políticas de seguridad para el buen uso del aplicativo.
  • Resolver dudas y/o inquietudes por parte del usuario.
  • Práctica (por parte del usuario) todo lo explicado en la capacitación con la ayuda de los desarrolladores.

Justificación

Es de vital importancia que el usuario use el aplicativo correctamente, conociendo los módulos que contiene, cómo es su utilización, las prevenciones que debe tener para no presentar inconvenientes al interactuar con el mismo, tener claridad la finalidad del aplicativo, qué hacer en caso de que el aplicativo no funcione de manera correcta.

Objetivos

  • Dar a conocer el aplicativo.
  • El usuario lo use correctamente.
  • Conocer la funcionalidad de cada parte del aplicativo.
  • Reducir el mal uso al aplicativo.
  • Incrementar los conocimientos en informática, mejorando sus habilidades para la manipulación del aplicativo.

Metodología

La capacitación se hará de manera presencial. Nosotros como desarrolladores del aplicativo daremos inducción al mismo, con lo cual plantemos llevarlo de esta manera:

  • Un miembro del equipo expondrá una presentación del aplicativo.
  • Se procede a resolver las preguntas surgidas durante las aplicativo.
  • Se procede a que cada uno de los usuarios, exponga si entendió claramente la información que se le presentó.
  • Haremos preguntas técnicas y generales, estas consisten en verificar la buena marcha del proceso del aprendizaje a través de preguntas y respuestas hábilmente encadenadas.

Dado que el restaurante no cuenta con el personal mayor a 10 personas, los dividiremos en dúos de tal forma en que 3 desarrolladores le mostrarán y explicarán de manera sencilla y clara las funcionales del software, sobre todo cada uno de los módulos en los que cada usuario tendrá acceso dependiendo del rol que se le es asignado.
